# 日本語文字カウントツール 詳細設計書

## 1. 概要
リアルタイムで文字数をカウントし、AIを活用して指定文字数に調整する機能を持つWebアプリケーション

## 2. 技術スタック
- **フロントエンド**: React 18 + TypeScript
- **スタイリング**: Tailwind CSS（GitHub風ダークテーマ）
- **状態管理**: React Hooks (useState, useEffect)
- **AI API**: Google Gemini API
- **ビルドツール**: Vite

## 3. 機能要件

### 3.1 文字カウント機能
- **リアルタイムカウント**: 入力と同時に自動カウント
- **3種類のカウント表示**:
  1. 総文字数（すべての文字を含む）
  2. 改行除外文字数（改行コードを除く）
  3. 改行・空白除外文字数（改行コードと空白文字を除く）

### 3.2 AI文章調整機能
- **目標文字数設定**: ユーザーが目標文字数を指定
- **文章調整**: Gemini APIを使用して文章を増減
- **調整オプション**:
  - 文章を増やす（詳細化）
  - 文章を減らす（要約）
  - 自動調整（目標文字数に最適化）

## 5. コンポーネント設計

### 5.1 コンポーネント構成
```
App.tsx
├── Header.tsx（ヘッダー）
├── CharacterCounter.tsx（文字数表示）
├── TextEditor.tsx（テキスト入力）
└── AIAdjustPanel.tsx（AI調整機能）
```

### 5.2 状態管理
- `text`: 入力テキスト
- `targetCount`: 目標文字数
- `isLoading`: API通信中フラグ
- `error`: エラーメッセージ

## 6. API設計

### 6.1 Gemini API統合
- **エンドポイント**: Google Generative AI REST API
- **モデル**: gemini-pro
- **主要機能**:
  - `/api/adjust-text`: 文章調整リクエスト
  - パラメータ: `text`, `targetCount`, `action`

## 8. セキュリティ考慮事項
- API キーは環境変数で管理
- クライアントサイドでAPIキーを露出させない
- 必要に応じてサーバーレス関数経由でAPI呼び出し

## 9. パフォーマンス最適化
- デバウンス処理でAPI呼び出し頻度を制御
- React.memoで不要な再レンダリング防止
- 大量テキスト処理時のパフォーマンス考慮
